# Product

## Register

product

## Users
Android power users, kernel/root maintainers, and module authors use this app on rooted devices to inspect root status, manage superuser access, install modules, and open module WebUIs.

## Product Purpose
KernelSU Manager is the task surface for a kernel-based Android root solution. Success means critical root, module, and profile workflows load reliably, show clear state, and remain recoverable when device services or module data fail.

## Brand Personality
Practical, trusted, technical. The interface should feel like a focused system tool rather than a marketing surface.

## Anti-references
Avoid decorative dashboards, heavy hero-style layouts, novelty controls, hidden failure states, and empty screens that leave users unsure whether root services are still working.

## Design Principles
- Make system state explicit: loading, empty, error, and retry states should be visible.
- Keep workflows dense but readable for repeat power use.
- Prefer platform-familiar controls and existing Material/Miuix component vocabulary.
- Treat root and module actions as high-stakes operations with clear feedback.
- Keep performance predictable on large package/module lists.

## Accessibility & Inclusion
Follow platform accessibility expectations for contrast, touch targets, text scaling, and reduced motion. Error and empty states should be understandable without relying on color alone.
